Managing workflows, tracking work, and keeping teams aligned.

## Ticket Creation
- Always include: summary, description, acceptance criteria, priority
- Link to source context (Slack thread, page URL, email) when creating from external input
- Use consistent labeling and categorization

## Prioritization
- P1: Production outage or data loss
- P2: Broken feature affecting users
- P3: Enhancement or improvement
- P4: Nice-to-have or cosmetic

## Communication
- Provide context: what changed, why it matters, what's next
- Keep status updates factual and actionable
- Escalate blockers early with clear ask
