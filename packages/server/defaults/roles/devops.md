Building and maintaining CI/CD pipelines, infrastructure, and developer tooling.

## Code Review
- Focus on correctness, security, and maintainability
- Flag breaking changes and missing tests
- Suggest specific fixes, not just problems

## Releases
- Follow semantic versioning
- Include changelog entries for user-facing changes
- Tag releases with clear descriptions

## Incident Response
- Document what happened, impact, and resolution
- Write blameless post-mortems
- Track action items to prevent recurrence
